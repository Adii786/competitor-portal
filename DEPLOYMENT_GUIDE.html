🚀 Competitor Intel Portal — Deployment Guide

Complete step-by-step guide to go from files to a live online portal in ~30 minutes

1
Set Up Supabase (Free Database + Auth)
~10 minutes · Free forever · No credit card needed
FREE
🌐
Create your free Supabase account
Go to supabase.com → Click "Start your project" → Sign up with GitHub or email → It's completely free (500MB database, 50,000 monthly active users on free plan)
🏗
Create a new project
Click "New Project" → Enter name: competitor-intel → Choose region: Southeast Asia (Singapore) for best Pakistan latency → Set a strong database password → Click "Create new project" → Wait ~2 minutes for it to spin up
🔑
Copy your API credentials
In your project dashboard → Click ⚙️ Settings (left sidebar) → Click API → Copy two things:

📋 Project URL: looks like https://abcdefgh.supabase.co
📋 anon public key: a long JWT token starting with eyJhbGci...

Keep these — you'll paste them in the portal config screen.
🗄
Run the database setup script
In Supabase → Click SQL Editor (left sidebar, looks like </>) → Click "New query" → Open the file supabase_setup.sql provided → Copy the entire contents → Paste into the editor → Click Run (or press Ctrl+Enter)

You should see "Success. No rows returned" — this created all your tables, security rules, and database policies.
💡 The SQL script creates: user profiles table, submissions table, product entries table, and Row Level Security so field staff can ONLY see their own data
📧
Configure email settings (optional but recommended)
In Supabase → Authentication → Providers → Email → You can turn off "Confirm email" for easier testing (users don't need to verify email). For production, keep it ON for security.
2
Host on GitHub Pages (Free Web Hosting)
~10 minutes · Free forever · Gets you a public URL
FREE
👤
Create a GitHub account (if you don't have one)
Go to github.com → Sign Up → Free account is all you need. GitHub is where your portal files will live.
📁
Create a new repository
After logging in → Click the + icon (top right) → "New repository" → Repository name: competitor-portal → Set to Public (required for free GitHub Pages) → Click "Create repository"
⚠ The repository must be Public for free GitHub Pages hosting. Your Supabase credentials are NOT in the HTML (users enter them on first use), so this is safe.
⬆
Upload your files
In your new repository → Click "uploading an existing file" link → Drag and drop these files from your computer:

📄 index.html (the portal)
📄 supabase_setup.sql (for reference)
📄 DEPLOYMENT_GUIDE.html (this file)

→ Scroll down → Click "Commit changes"
🌍
Enable GitHub Pages
In your repository → Click ⚙️ Settings tab → Scroll down to Pages section (left sidebar) → Under "Source" → Select Deploy from a branch → Branch: main → Folder: / (root) → Click Save

Wait 2-3 minutes → Your portal is now live at:
https://YOUR-GITHUB-USERNAME.github.io/competitor-portal/
Share this URL with your entire team. It works on mobile and desktop.
🔗
Update your files later
If you need to update the portal → Go to your GitHub repository → Click on index.html → Click the ✏️ pencil icon → Edit → Commit changes. GitHub Pages automatically redeploys within 2 minutes.
3
First-Time Portal Configuration
~5 minutes · One-time setup per browser
⚙️
Connect the portal to Supabase
Open your GitHub Pages URL → You'll see a "First-Time Setup" screen → Paste your Supabase Project URL and anon public key → Click "Save & Connect"

This only happens once per device/browser. The credentials are saved locally.
👑
Create your admin (Head Office) account
On the login screen → Click "Create Account" tab → Fill in your details → Role: Head Office / Admin → Submit

Then go to Supabase → SQL Editor → Run this command to make yourself admin:
-- Replace with your actual email: UPDATE public.profiles SET role = 'ho' WHERE email = 'your@email.com';
👥
Share with field team
Send the portal URL to your field team. Each person:
① Opens the URL on their phone/laptop
② Pastes Supabase credentials (you share these once via WhatsApp)
③ Creates their own account (name, role=Field, town, email, password)
④ Done — they can immediately start entering data
💡 Alternatively, you can create accounts for all field staff yourself in Supabase → Authentication → Users → "Invite user"
4
Understanding Roles & Privacy
Who sees what
🔐
Role-based access control
RoleCan SeeViews Available
Field TSS/TSMOnly their OWN submissionsEnter Data, My Reports
SupervisorAll submissions + dashboardDashboard, All Submissions, New Intel, My Reports
RSM / ManagerAll submissions + dashboardDashboard, All Submissions, New Intel
Head OfficeEverythingAll views + User Management

Privacy is enforced at the database level (Supabase Row Level Security) — not just the UI. Even if someone tries to access data directly, the database will block them.
🔒
Change a user's role in Supabase
Go to Supabase → SQL Editor → Run:
-- Options: 'field', 'supervisor', 'rsm', 'ho' UPDATE public.profiles SET role = 'rsm' WHERE email = 'arif.saeed@dollarpk.com';
5
Optional: Custom Domain (e.g. intel.dollarpk.com)
~15 minutes · Costs ~$10–15/year for domain
Optional
🌐
Buy a domain (if you don't have one)
Buy from namecheap.com or godaddy.com — about $10–15/year. Example: dollarpk.com or dollar-intel.com
⚙️
Connect domain to GitHub Pages
In your domain registrar → Add a CNAME record: intel → YOUR-USERNAME.github.io

In GitHub → Repository Settings → Pages → Custom domain → Enter intel.dollarpk.com → Save → Enable "Enforce HTTPS"

Wait 10–30 minutes for DNS propagation. Your portal will then be available at https://intel.dollarpk.com

🆘 Troubleshooting Common Issues

Page shows blank: Wait 5 min after enabling GitHub Pages

"Connection failed": Check your Supabase URL and key are correct

"Invalid login": Check email/password. Reset via Supabase Auth panel

Can't see other submissions (HO): Make sure your role is set to 'ho' in profiles table

SQL errors: Make sure you ran the full supabase_setup.sql script
